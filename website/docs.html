Documentation

Becket is a local intelligence layer that combines a verified code graph, a graph-grounded knowledge wiki, and context assembly that returns real source snippets for AI agents and developers.

Three Knowledge Layers

Becket indexes your codebase locally and exposes three coordinated layers: a deterministic graph (ground truth), a symbol-anchored wiki (intent and gotchas), and context assembly (real snippets within a token budget).

Context AssemblyCode snippets + wiki + impact, token-budgeted
Grounded Repo WikiMarkdown pages anchored to symbol IDs, MCP-authored, lint-checked
Deterministic CoreSymbols, graph, flows, impact — from tree-sitter, no LLM

CLI Reference

Initialize analysis

terminal
becket build

Generates versioned JSON artifacts under .becket/:

.becket/
  architecture.json
  symbols.json
  flows.json
  dependencies.json
  entrypoints.json
  wiki/
    index.md
    *.md

Knowledge wiki

becket wiki sync
becket wiki lint
becket wiki show OrderService

Sync creates/updates grounded markdown pages (lazy, via MCP host). Lint flags stale or contradictory pages against the live graph.

Query impact

terminal
becket impact UserService

Returns affected modules, downstream dependencies, related tests, and potential risk zones.

Understand a flow

terminal
becket flow payment

Shows end-to-end execution path, service interactions, and external systems.

Context bundle (code included)

terminal
becket context PaymentService --format md --budget 6000 --task fix

Returns wiki page + real code snippets + impact set, packed within the token budget. Enough source to fix a bug — not the whole repo.

Domain refinement (optional)

becket domain rename <auto-id> payment
becket domain add payment src/billing/** PaymentService

Zero-config by default. Refine auto-discovered domains via CLI — no config file required.

Grounded Repo Wiki

Markdown pages for modules, services, and flows — each anchored to real symbol IDs from the deterministic graph. Structure is compiled from the graph; optional prose (intent & gotchas) via MCP. Stale claims are flagged automatically.

  • Pages authored lazily via MCP sampling (host model, no bundled LLM)
  • graph_fingerprint detects when anchored code changed
  • wiki lint compares page claims to live edges
  • index.md routes agents to relevant pages first

MCP Integration

Becket exposes an MCP server over stdio for seamless agent integration.

becket-mcp
Tool Description
get_contextCode snippets + wiki page + impact, packed to budget
get_wikiGrounded markdown page or index router
get_impactChange impact analysis
get_flowBusiness flow reconstruction
get_dependenciesDirect and transitive dependencies

Wiki authoring and text enrichment use MCP sampling — the host agent's model handles prose. Becket ships no LLM, holds no API keys, and never generates code — only slices real source.

Design Principles

Deterministic First

Core analysis is deterministic. Structure is derived from code, not guessed.

AI-Augmented, Not AI-Dependent

AI enhances interpretation via MCP sampling. The tool works fully without it.

Local-First

All analysis runs locally. Privacy, speed, reproducibility. Zero telemetry.

Machine-Readable Outputs

JSON-compatible, stable schema, versioned artifacts.

Graph-Grounded Wiki

Wiki pages anchor to symbol IDs and are lint-checked against the live graph. The graph is always ground truth.

Architecture

Rust-first deterministic core, graph-grounded wiki, context assembly, SQLite index, MCP server. Multi-repo workspaces and cross-service linking.

Quick start

Three steps to wire Becket into your agent workflow.

  1. Install becket-cli and becket-mcp (npm or Cargo).
  2. Run becket build in your repository root.
  3. Add becket-mcp to Cursor or Claude Code — the agent calls get_context before edits.

Cursor MCP config

.cursor/mcp.json
{
  "mcpServers": {
    "becket": {
      "command": "becket-mcp",
      "env": {
        "BECKET_ROOT": "${workspaceFolder}"
      }
    }
  }
}

Installation

Becket is 100% free and open source (Apache-2.0). No account, no subscription, no telemetry.

npm

terminal
npx becket build

Cargo

terminal
cargo install becket

After install, run becket build in any repository. Zero configuration required.
